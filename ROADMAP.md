# Roadmap

This roadmap describes direction, not a promise or fixed release schedule.
Issues own implementation details and acceptance criteria.

## Now — establish the original identity

- Create the first flagship original fighter with 50 purpose-built animation
  slots and a competitively coherent move set.
- Add a second original stage and improve the public roster's visual
  consistency.
- Tune hitboxes, knockback, recovery, movement, and defensive options through
  repeatable playtests.
- Make the fighter and stage starter workflows usable in minutes from a fresh
  clone.
- Grow a welcoming contributor workflow around Issues and Discussions.

## Next — make creation dramatically easier

- Preview draft fighter and stage packs without publishing them to the roster.
- Add editor schemas and actionable, targeted validation for render manifests.
- Generate stage thumbnails and checksums from the command line.
- Ship small, redistributable starter packs for fighter and stage authors.
- Publish versioned pack contracts and migration notes.

## Later — expand competitive and community play

- Add replay and deterministic match-debugging tools.
- Explore rollback-capable online multiplayer after deterministic simulation is
  demonstrated and tested.
- Add community pack discovery without allowing executable content.
- Formalize competitive balance targets and publish playtest data.

## Not currently planned

- Distribution of third-party franchise content.
- Executable scripts inside content packs.
- A marketplace or monetized asset store.
